## 1. Общая концепция

**BOATTOMORROW** — двусторонняя контент-платформа для яхтенной индустрии.

Поставщики (чартерные компании, производители яхт, дистрибуторы, яхтенные школы) через интерактивный AI-опросник создают SEO-оптимизированный контент о дестинейшенах, маршрутах, яхтах и обучении. Этот контент публикуется на публичном сайте для яхтинг-энтузиастов, которые могут оставить заявку на подбор яхты прямо со страницы статьи.

### Ключевые стейкхолдеры

| Роль | Кто | Что делает на платформе |
|------|-----|------------------------|
| Поставщик контента | Чартерные компании | Описывают дестинейшены, маршруты, свой флот |
| Поставщик контента | Производители и дистрибуторы яхт | Описывают модели, технологии, преимущества |
| Поставщик контента | Яхтенные школы | Описывают программы обучения, локации, курсы |
| Потребитель контента | Яхтсмены и туристы | Читают, оставляют заявки на подбор яхты |
| Администратор | Команда BOATTOMORROW | Модерируют контент, управляют платформой |

### Основной сценарий работы

1. Поставщик регистрируется, выбирает свой тип (чартер / производитель / школа)
2. Проходит AI-квиз, адаптированный под его тип
3. Загружает фотографии
4. LLM генерирует SEO-статью и профиль компании
5. Статья публикуется на сайте
6. Читатель оставляет заявку на подбор → лид уходит поставщику

## 2. Типы поставщиков контента

Платформа изначально проектируется под три типа поставщиков. Тип выбирается при регистрации и определяет: структуру профиля, сценарий AI-квиза и рубрики, в которых публикуется контент.

### 2.1. Charter Company (чартерная компания)

**Профиль включает:** название, страны присутствия, типы яхт в флоте, сезонность, языки общения, сертификаты, контактные данные.

**Публикует контент в рубриках:**
- Destinations — описания дестинейшенов
- Routes — яхтенные маршруты
- Tips — советы по яхтингу в регионе

**Лидогенерация:** читатель оставляет заявку «Хочу яхту в этой дестинации» → лид идёт в эту компанию (или в общий пул, если статья написана несколькими).

### 2.2. Boat Manufacturer / Distributor (производитель или дистрибутор яхт)

**Профиль включает:** бренды в портфеле, типы судов (парусные / моторные / катамараны), регионы дистрибуции, шоурумы, контактные данные.

**Публикует контент в рубриках:**
- Boats — обзоры и описания моделей яхт
- Gear & Tech — оборудование, технологии, инновации
- Buying Guide — гайды по выбору яхты

**Лидогенерация:** читатель оставляет заявку «Хочу узнать цену / договориться о тест-драйве».

### 2.3. Sailing School (яхтенная школа)

**Профиль включает:** сертифицирующие организации (IYT, RYA, ASA...), уровни обучения, локации школы, форматы курсов (онлайн / очно / на воде), языки обучения.

**Публикует контент в рубриках:**
- Learning — статьи об обучении, курсах, квалификациях
- Destinations — учебные маршруты и базы (пересечение с чартерами)
- Tips — советы для начинающих яхтсменов

**Лидогенерация:** читатель оставляет заявку «Хочу записаться на курс».

### 2.4. Расширяемость рубрикатора

Архитектура предусматривает добавление новых типов поставщиков без переработки ядра:
- Таблица `SupplierType` в БД хранит типы и их конфигурацию
- Квиз и профиль собираются из JSON-конфига по типу
- Рубрики и их маппинг на типы — в таблице `Category`

## 3. Техническая архитектура

### 3.1. Почему не WordPress

WordPress не используется, потому что:
- Сложная кастомизация под AI-квиз и динамическую генерацию контента
- Слабая нативная интеграция с LLM API
- Claude Code не оптимизирован для WP-разработки
- Избыточность плагинов, технический долг, уязвимости

### 3.2. Рекомендуемый стек

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| Framework | Next.js 14 (App Router) | SSR/SSG для SEO, React UI, API Routes в одном проекте |
| Стили | Tailwind CSS | Быстрая адаптивная верстка |
| База данных | PostgreSQL (Supabase) | Реляционные данные, встроенный Auth, Storage |
| ORM | Prisma | Типобезопасный доступ к БД, миграции |
| Аутентификация | NextAuth.js | Email/пароль, extensible под OAuth |
| Хранение файлов | Supabase Storage | Загрузка и CDN-раздача фотографий |
| AI-квиз | Anthropic Claude API | Генерация статей и профилей, диалоговый квиз |
| SEO | next-seo + next-sitemap | Мета-теги, OpenGraph, sitemap.xml |
| Email | Resend | Уведомления, верификация, лиды |
| Деплой | Vercel + Supabase | Serverless, автоскейлинг, CI/CD из Git |

## 5. Функциональные модули

### 5.1. Публичный сайт

#### Главная страница (/)
- Hero с поиском по дестинейшенам и типу запроса
- Навигация по рубрикам (Destinations / Boats / Learning / Routes / Tips)
- Блок «Популярные регионы» с карточками
- Блок «Недавние статьи» — универсальная лента
- Блок «Наши поставщики» — карточки компаний с лого и типом
- Счётчики: статей, дестинейшенов, поставщиков

#### Страница рубрики (/destinations, /boats, /learning, ...)
- Фильтры, специфичные для рубрики:
  - Destinations: регион, сезон, тип яхты
  - Boats: тип судна, бренд, длина
  - Learning: уровень, сертификат, формат, регион
- Сортировка: новые / популярные / с заявками
- Пагинация

#### Страница статьи (/articles/[slug])
- Полный текст в MDX-рендере
- Галерея фотографий (lightbox)
- **Блок лидогенерации** — форма заявки (подробно в разделе 7)
- Sidebar: профиль автора-поставщика, похожие статьи
- JSON-LD: Article + BreadcrumbList
- OG-теги для шеринга

#### Страница профиля поставщика (/suppliers/[slug])
- Обложка, лого, название, тип поставщика
- Описание (из профиля, Markdown)
- Специфичные данные (регионы, флот / бренды / программы)
- Галерея
- Список опубликованных статей этого поставщика
- **Форма заявки** (адресована напрямую поставщику)

## 6. AI-квиз — центральный модуль

Квиз реализован как чат-интерфейс. LLM ведёт диалог, задаёт вопросы в зависимости от типа поставщика и цели (статья или профиль). Прогресс сохраняется в `QuizSession` — квиз можно продолжить позже.

### 6.1. Квиз для статей чартерной компании

| Шаг | Вопрос LLM | Тип |
|-----|-----------|-----|
| 1 | Тип контента: дестинейшен / маршрут / совет | Выбор |
| 2 | Название места/региона | Текст |
| 3 | Что делает это место особенным для яхтсмена? | Текст |
| 4 | Лучший сезон, ветра, особенности погоды | Текст |
| 5 | Марины и якорные стоянки | Текст |
| 6 | Что интересного на берегу | Текст |
| 7 | Типы яхт для этого маршрута | Текст |
| 8 | Для кого этот дестинейшен (аудитория) | Текст |
| 9 | Инсайдерский совет от капитана | Текст |
| 10 | Загрузка фото (3–10 шт.) | Upload |

### 6.2. Квиз для статей производителя / дистрибутора

| Шаг | Вопрос LLM | Тип |
|-----|-----------|-----|

| 2 | Модель или категория яхты | Текст |
| 3 | Для кого эта яхта — опыт, задачи, регионы | Текст |

| 5 | Что выделяет на фоне конкурентов | Текст |

| 7 | Ценовой диапазон и что входит | Текст |
| 8 | Загрузка фото яхты | Upload |

### 6.3. Квиз для статей яхтенной школы

| Шаг | Вопрос LLM | Тип |
|-----|-----------|-----|
| 1 | Тип контента: обзор курса / дестинейшен обучения / совет | Выбор |
| 2 | Название курса или программы | Текст |
| 3 | Для кого: уровень, возраст, опыт | Текст |

| 9 | Контактная информация | Текст |
| 10 | Логотип + обложка + галерея | Upload |

### 6.5. Промпт генерации статьи

После завершения квиза отправляется итоговый промпт:

```
SYSTEM:
Ты — профессиональный SEO-копирайтер яхтенного журнала.
Пиши живо, экспертно, для людей которые любят море.

USER:
На основе ответов поставщика [type: charter | manufacturer | school]
напиши статью в рубрику [category].

Данные из квиза:
[JSON с ответами]

Требования к статье:
- H1 (до 60 симв., с ключевым словом дестинейшена/продукта)
- Вступление 2–3 абзаца
- 4–6 секций с H2-заголовками (по теме)
- Заключение с призывом к действию
- metaTitle (60 симв.)
- metaDescription (160 симв.)
- tags: 5 штук
- slug (латиница, через дефис)
- altTexts для каждого загруженного фото

Ответ строго в JSON:
{
  "title": "...",
  "slug": "...",
  "content": "...(Markdown)...",
  "metaTitle": "...",
  "metaDescription": "...",
```

## 7. Лидогенерация

Это критически важный модуль, напрямую монетизирующий платформу для поставщиков.

### 7.1. Точки входа для лида

| Где размещена форма | Intent | Адресат лида |
|--------------------|--------|-------------|
| Страница статьи (блок после текста) | По типу статьи | Автор статьи |
| Профиль поставщика | По типу поставщика | Этот поставщик |
| Главная страница (общая форма) | general | Пул активных поставщиков |

### 7.2. Форма заявки — структура

Форма адаптируется под intent:

**charter_booking (яхтенный чартер)**
```
Имя*
Email*
Телефон
Дестинация / регион интереса (prefilled из статьи)
Примерные даты
Количество человек
Дополнительные пожелания
[Подобрать яхту]
```

**boat_purchase (покупка яхты)**
```
Имя*
Email*
Телефон
Интересующая модель (prefilled из статьи)
Бюджет (опционально)
Вопрос или запрос
[Узнать цену / Договориться о тест-драйве]
```

**school_enrollment (запись в школу)**
```
Имя*
Email*
Телефон
Интересующий курс (prefilled из статьи)
Уровень подготовки
Предпочтительные даты
[Записаться на курс]
```

### 7.3. Обработка лида

1. Лид сохраняется в таблицу `Lead` со статусом `new`
2. Поставщику отправляется email (Resend) с деталями заявки
3. В кабинете поставщика появляется уведомление
4. Счётчик `leadsCount` на статье инкрементируется
5. Суперадмин видит все лиды в общем дашборде

### 7.4. Кабинет лидов у поставщика

Раздел `/supplier/leads` в кабинете поставщика:
- Список лидов с фильтрами (статус, дата, источник)
- Смена статуса: new → seen → contacted → closed
- Экспорт в CSV

### 7.5. UX-детали формы

- Форма появляется как sticky-блок на десктопе (после первого экрана статьи)
- На мобильном — фиксированная кнопка внизу экрана → открывает modal
- После отправки: «Заявка отправлена! Поставщик свяжется с вами в течение 24 часов»
- Защита от спама: honeypot-поле + rate limit (3 заявки с IP за 10 минут)
- Поля `destination` и `courseName` / `modelName` предзаполняются из контекста статьи

### 9.1. Роли и права

| Действие | Гость | Supplier | Superadmin |
|----------|-------|----------|-----------|
| Читать статьи | ✅ | ✅ | ✅ |
| Искать, фильтровать | ✅ | ✅ | ✅ |
| Просматривать профили поставщиков | ✅ | ✅ | ✅ |
| Оставлять заявки | ✅ | ✅ | ✅ |
| Проходить AI-квиз | ❌ | ✅ | ✅ |
| Управлять своими статьями | ❌ | ✅ | ✅ |
| Управлять своим профилем | ❌ | ✅ | ✅ |
| Видеть свои лиды | ❌ | ✅ | ✅ |
| Модерировать все статьи | ❌ | ❌ | ✅ |
| Управлять поставщиками | ❌ | ❌ | ✅ |
| Видеть все лиды | ❌ | ❌ | ✅ |

### 9.2. Регистрация поставщика

1. Выбор типа поставщика (`charter` / `manufacturer` / `school`)
2. Email + пароль + название компании
3. Верификация email (ссылка через Resend)
4. После верификации — статус `pending` или `active` (настраивается)
5. Если `pending` — письмо суперадмину, автоматически или вручную одобряет

### 9.3. Меры безопасности

- Пароли: bcrypt, salt rounds 12
- JWT: httpOnly cookies, не localStorage
- CSRF: встроен в NextAuth
- Rate limiting: API квиза — 10 req/min, форма лидов — 3 req/10min per IP
- Валидация загружаемых файлов: MIME-type на сервере, max 10MB, только JPEG/PNG/WebP
- Все `/supplier/*` и `/admin/*` маршруты защищены middleware
- Поставщик видит только свои данные (проверка `supplierId` в каждом запросе)
- HTTPS: Vercel обеспечивает автоматически

## 10. SEO-оптимизация

### 10.1. Технический SEO

- **SSR/SSG** для всех статей и профилей — контент индексируется поисковиками
- **ISR** (Incremental Static Regeneration) для статей — revalidate каждые 24 часа
- **sitemap.xml** — автогенерация через next-sitemap при публикации
- **robots.txt** — закрыть `/supplier`, `/admin` от индексации
- **Canonical URLs** — предотвращение дублей при фильтрации
- **Core Web Vitals** — оптимизация через Next.js Image (lazy loading, WebP)

### 10.2. Структурированные данные (JSON-LD)

На странице статьи:
```json
{
  "@type": "Article",
  "headline": "...",
  "author": { "@type": "Organization", "name": "...", "url": "..." },
  "publisher": { "@type": "Organization", "name": "BOATTOMORROW" },
  "image": "...",
  "datePublished": "..."
}
```

На профиле поставщика:
```json
{
  "@type": "TravelAgency | Store | EducationalOrganization",
  "name": "...",
  "description": "...",
  "areaServed": [...]
}
```

### 10.3. Контентный SEO (генерируется LLM)

- H1 содержит ключевое слово (название дестинейшена/яхты/курса + «яхтинг»)
- metaTitle (≤60 симв.) с ключевым словом
- metaDescription (≤160 симв.) с призывом
- Slug: латиница, через дефис (например `catamaran-sailing-croatia`)
- Alt-тексты для каждого фото
- Внутренняя перелинковка: похожие статьи в sidebar
